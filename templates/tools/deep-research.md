## Research Intake

- Primary source registry file: `DATA_SOURCES.md` in the current workspace.
- Load `DATA_SOURCES.md` before launching multi-stream research.
- If the source list does not cover the ask, pause and request an update.
- Capture:
  - objective
  - success criteria
  - constraints
  - deadline
  - required output format
- If any are missing, ask before deep execution.

## Execution Framework

1. Define research hypotheses and key questions.
2. Break work into streams (market, product, technical, operations, etc.).
3. Assign streams to parallel sub-tasks when available.
4. Collect evidence with citations and confidence ratings.
5. Synthesize into recommendations and execution plan.

## Source Strategy

- Prefer primary sources first (official docs, first-party data, direct APIs).
- Use secondary sources only for triangulation.
- Track citation links inline with each major claim.

## Output Structure

1. Executive summary.
2. Detailed findings by stream.
3. Tradeoffs and risks.
4. Prioritized recommendations.
5. 30/60/90-day action plan.

## Guardrails

- Do not optimize for speed at the cost of evidence quality.
- Surface contradictory evidence explicitly.
- Keep recommendations actionable and measurable.
